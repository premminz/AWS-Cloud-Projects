# AWS Identity and Access Management (IAM)

## Overview

This section contains my hands-on AWS IAM projects and practical implementations. The purpose of these labs is to understand how AWS manages authentication, authorization, and access control for cloud resources.

Through these projects, I explored IAM users, groups, policies, roles, and security best practices such as Multi-Factor Authentication (MFA).

---

## Projects Included

### 1. Create IAM User

Created IAM users with AWS Management Console access and programmatic access.

### 2. Create IAM Policy

Created custom IAM policies to provide fine-grained permissions to AWS resources.

### 3. Group Creation and User Access Management

Created IAM groups and assigned permissions to multiple users through group-based access control.

### 4. Multi-Factor Authentication (MFA)

Enabled MFA for IAM users to enhance account security and protect against unauthorized access.

## Objective

The objective of these IAM projects is to develop practical expertise in AWS identity and access management by implementing real-world security configurations. Through these hands-on labs, I gained experience in creating and managing IAM users, groups, policies, and multi-factor authentication while following AWS security best practices.

As a Network Engineer transitioning into Cloud Engineering, these projects helped me strengthen my understanding of cloud security, access control, and resource management, which are essential skills for designing and operating secure AWS environments.
